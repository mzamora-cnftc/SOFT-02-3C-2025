Práctica 02: Relaciones entre Clases en Situaciones Cotidianas

Objetivo

Desarrollar la capacidad de identificar y modelar relaciones entre clases (asociación, agregación, composición) en situaciones de la vida real, implementando una solución en Java con interfaz de usuario básica.

Instrucciones

Parte 1: Análisis y Diseño (30%)

  1. Selección de Escenario:

    • Cada estudiante/grupo debe elegir UNA situación cotidiana de su preferencia
    • Ejemplos sugeridos (pero no limitados a):
      • Sistema de gestión de restaurante
      • Aplicación de delivery de comida
      • Gestión de gym/fitness
      • Sistema de reservas de hotel
      • Plataforma de cursos online
      • App de transporte

  2. Identificación de Relaciones:

    • Debe incluir al menos 2 tipos de las siguientes relaciones:
      • Asociación
      • Agregación
      • Composición

  3. Diagrama de Clases:

    • Crear un diagrama UML que muestre las clases, atributos, métodos y relaciones identificadas

Parte 2: Implementación en Java (50%)

Requisitos técnicos:

  • Mínimo 3 clases relacionadas
  • Implementar menú interactivo por consola
  • Demostrar las relaciones mediante funcionalidades específicas
  • Validaciones básicas de entrada

Estructura del menú:

// Ejemplo de estructura de menú requerida
public void mostrarMenu() {
    System.out.println("1. Crear [objeto]");
    System.out.println("2. Listar [objetos]"); 
    System.out.println("3. Realizar [acción que demuestre relación]");
    System.out.println("4. Salir");
}

Parte 3: Documentación (20%)

Informe breve que incluya:

  • Descripción del escenario elegido
  • Justificación de cada relación identificada
  • Ejemplos de cómo se demuestran las relaciones en el código
  • Conclusiones del aprendizaje

Rúbrica de Evaluación

Criterio Puntos Descripción
Análisis de relaciones 30% Correcta identificación y justificación de relaciones
Implementación Java 40% Código funcional, menú interactivo, relaciones demostradas
Diagrama de clases 20% Diagrama UML claro y correcto
Documentación 10% Informe claro y completo

Preguntas Frecuentes

❓ ¿Cuántas relaciones debo implementar como mínimo?

  • Mínimo 2 tipos diferentes de relaciones.

❓ ¿El menú debe ser muy complejo?

  • No, pero debe permitir interactuar y demostrar las relaciones.
